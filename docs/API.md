# API Documentation

Base URL (local): `http://127.0.0.1:8000`

Interactive Swagger UI is auto-generated by FastAPI at `/docs` (and
ReDoc at `/redoc`) once the server is running:

```bash
uvicorn api:app --reload
```

All request bodies are `multipart/form-data` (the resume is an uploaded
file). All responses are JSON.

---

## `POST /analyze`

Runs the full pipeline: parse resume → analyze JD → compute ATS score →
generate improvements → (optionally) generate a cover letter → export
files → save to database.

**Form fields**

| Field | Type | Required | Notes |
|---|---|---|---|
| `resume` | file | yes | PDF, DOCX, or TXT |
| `job_description` | text | yes | Raw job description text |
| `company_name` | text | no | Enables cover letter generation |
| `job_title` | text | no | Enables cover letter generation |
| `hiring_manager` | text | no | |
| `generate_cover_letter` | bool | no | Default `true` |
| `output_dir` | text | no | Default `exports` |

**Example**

```bash
curl -X POST http://127.0.0.1:8000/analyze \
  -F "resume=@examples/example_resume.txt" \
  -F "job_description=$(cat examples/example_job_description.txt)" \
  -F "company_name=Meridian Cloud Technologies" \
  -F "job_title=Senior Backend Engineer"
```

**Response** — a `PipelineResult` JSON object:

```json
{
  "parsed_resume": { "...": "ParsedResume" },
  "job_requirements": { "...": "JobRequirements" },
  "ats_report": {
    "ats_score": 72.5,
    "match_breakdown": {
      "keyword_match": 70.0, "skills_match": 80.0, "experience_match": 60.0,
      "project_match": 50.0, "education_match": 100.0, "formatting_score": 90.0
    },
    "matched_keywords": ["Python", "Docker"],
    "missing_keywords": ["Kubernetes", "Terraform"],
    "strengths": ["..."],
    "weaknesses": ["..."],
    "summary": "..."
  },
  "improved_resume": { "...": "ImprovedResume" },
  "cover_letter": { "...": "CoverLetter or null" },
  "exported_files": {
    "markdown_report": "exports/jordan_ellis_ats_report.md",
    "pdf_report": "exports/jordan_ellis_ats_report.pdf",
    "optimized_resume_docx": "exports/jordan_ellis_optimized_resume.docx",
    "cover_letter_docx": "exports/jordan_ellis_cover_letter.docx"
  },
  "record_id": 1
}
```

`422 Unprocessable Entity` if the job description is too short (< 50 chars)
or the resume has no extractable text.

## `POST /improve`

Parse + analyze + improve only — no export, no database write, no cover
letter. Useful for a "preview improvements" UI flow.

**Form fields**: `resume` (file), `job_description` (text).

**Response**: an `ImprovedResume` JSON object.

---

## `POST /cover-letter`

**Form fields**: `resume` (file), `job_description` (text), `company_name`
(text, required), `job_title` (text, required), `hiring_manager` (text,
optional).

**Response**: a `CoverLetter` JSON object (`full_text` is ready to send
as-is).

---

## `GET /history`

**Query params**: `limit` (default 50), `search` (matches candidate name,
job title, or company name).

**Response**: array of `AnalysisRecord`:

```json
[
  {
    "id": 1,
    "candidate_name": "Jordan Ellis",
    "job_title": "Senior Backend Engineer",
    "company_name": "Meridian Cloud Technologies",
    "ats_score": 72.5,
    "resume_path": "...",
    "jd_path": null,
    "report_path": "exports/jordan_ellis_ats_report.md",
    "created_at": "2026-07-19T12:00:00+00:00"
  }
]
```

---

## `GET /report/{id}`

**Response**: the full stored row, including the complete `ATSAnalysisReport`
under `report_json`. `404` if the id doesn't exist.

---

## `GET /health`

Liveness check: `{"status": "ok"}`.
